# 打断执行

任务跑偏了、点错了应用，或者你只是想换个指令重新开始时，不必等它自己跑完。本指南介绍如何中止正在执行的任务，以及如何重置对话从头来过。

## 中止正在执行的任务

任务一旦开始执行，对话框右下角原本的发送按钮会自动变成红色的停止按钮（方块图标）。

![任务执行中](/img/screenshots/chat-running.png)

点击这个停止按钮即可中止当前任务。后端在收到请求后会停止下一步动作，通常在 1 秒内生效，按钮会短暂显示加载状态，中止完成后界面提示「对话已中止」。

中止只会停下当前这一次执行，已经完成的步骤和聊天记录都会保留下来，你可以接着发送新的指令继续操作。如果中止失败，会看到「中止对话失败」的提示，此时重试一次即可。

## 两种模式都支持中止

无论你用的是[经典模式还是分层代理模式](../explanation/modes.md)，停止按钮的位置和操作都完全一致，你不需要关心当前跑的是哪种 agent。

区别只在底层：经典模式通过 `POST /api/chat/abort` 中止当前对话，分层代理模式通过 `POST /api/layered-agent/abort` 中止规划与执行循环。前端会根据当前会话自动选择正确的接口，你只管点停止按钮就行。

## 重置对话从头开始

中止解决的是「停下这一次」，而当你想彻底换个话题、清空当前上下文重新开始时，使用「重置对话」。

点击对话区上方的「重置对话」按钮，当前会话的上下文（历史消息、待发送的附件等）会被清空，回到刚初始化设备时的干净状态。重置不影响已连接的设备和已配置的模型，你可以立刻输入新指令开始一个全新的任务。

如果任务还在执行，请先点停止按钮中止，再重置对话。

## 相关接口

上面提到的两个中止接口的请求参数和返回值，可以在 [REST API 参考](../reference/rest-api.md) 中查阅：

- `POST /api/chat/abort`：中止经典模式当前对话。
- `POST /api/layered-agent/abort`：中止分层代理模式当前任务。
