# 使用 Web 终端执行 ADB 命令

Web 终端让你在浏览器里直接执行 ADB 命令，不必切到本地命令行。它运行在 ADB-only 模式下，只接受 `adb` 开头的命令，适合临时排查设备连接、查看进程或快速验证一条 shell 指令。

本指南假设你已经启动了后端、并能在浏览器里打开前端界面。如果还没连上设备，先看[连接设备](./connect-device.md)。

## 启用 Web 终端

出于安全考虑，Web 终端默认只对本机（loopback）开放。如果后端跑在远程主机，或者你通过非 `127.0.0.1` 的地址访问，需要显式开启：

```bash
export AUTOGLM_ENABLE_WEB_TERMINAL=1
```

设置后重启后端即可生效。未开启时，从非本机来源访问会收到 403，WebSocket 也会被拒绝。环境变量的完整说明见[环境变量参考](../reference/env-vars.md)。

## 打开终端并执行命令

在侧边栏进入「终端」页面。页面加载时会自动创建一个会话并连上 WebSocket，初始化完成后终端会提示已就绪，目前只允许执行 adb 命令。

![Web 终端](/img/screenshots/terminal.png)

光标停在 `adb>` 提示符后，直接输入命令并回车，例如：

```
adb devices
adb connect 192.168.1.10:5555
adb shell getprop ro.product.model
```

输出会实时回显在终端里。每条命令执行完会附带一行退出码，方便你判断是否成功。

## 查看会话信息与管理会话

左侧面板展示当前会话的运行情况：

- 状态：会话从 `created`、`starting` 到 `running`，关闭后变为 `closed`。
- 会话 ID 与启动命令：用于核对当前连接的是哪个会话。
- Socket：显示 WebSocket 是已连接还是未连接。

面板下方有三个操作：

- 重建会话：关闭当前会话并新建一个，同时清空屏幕。会话异常或想要干净环境时用它。
- 清屏：只清空终端显示，不影响会话本身。
- 关闭会话：结束当前会话并断开连接。此后界面会提示没有活动会话，可随时重新创建。

离开终端页面时，会话会自动关闭，不会在后台残留。

## 使用快捷命令

「快捷命令」区域帮你省去手敲常用命令：

- 列出设备：执行 `adb devices -l`，列出所有已连接设备及其属性。
- 设备列表：每个已连接设备是一个按钮，点击即对该设备执行 `adb -s <序列号> shell`，进入它的交互式 shell。

如果列表为空会提示当前没有已连接设备。点右上角的刷新按钮可以重新拉取设备列表。

## 安全说明

Web 终端刻意限制了能力范围，避免暴露任意命令执行：

- 只接受 `adb` 开头的命令，其他命令会被拒绝并提示 `Only adb commands are allowed.`。
- 内置少量辅助命令：`help`（或 `?`）查看用法、`clear` 清屏、`exit` / `quit` 退出会话。
- 非本机访问必须设置 `AUTOGLM_ENABLE_WEB_TERMINAL=1` 才能启用，否则一律拒绝。

请把它当作受限的排查工具，而不是完整的远程 shell。如果命令报错或会话连不上，参考[常见问题](./troubleshooting.md)排查。
